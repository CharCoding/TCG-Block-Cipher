Obfuscated JavaScript Private Variable

This monstrosity is an attempt at hiding a secret key from the client. It returns a function t(f) that takes one argument and checks it against the key. Only if the argument matches the key will the function return true. t(f) happens to be an asynchronous function, so you can print its result by running t("secretKeyHiddenOnTheClient").then(console.log); // --> false.

This seems not possible at first, since everything must be loaded into the browser for it to run. However, even if you deobfuscate the 7-character obfuscation (adapted from jscrew.it), you will not find that secret key. A few calls are made to crypto.getRandomValues and crypto.subtle.digest, which reconstructs the key (not any random key) in real time from random numbers. The random numbers, the digest, and other helper functions are then immediately hidden by setting t.__proto__ to null and freezing the function. The whole thing is obfuscated mainly to avoid breakpoints being placed at these important steps.

The motivation of this project is to make a break-my-cipher challenge where the secret key can be used for encrypting/decrypting arbitrary texts, but the client should not be able to see the key directly (the goal is to derive the key with a chosen-plaintext attack). The goal would be to make it harder to derive the key by going through the script above than doing a cryptanalysis on the actual cipher challenge. Of course, it would be much easier to just host a server that encrypts/decrypts messages without disclosing any information of the key, but I do not know how to run servers.

This project is far from perfect; you can still theoretically derive the key by going through the mess step by step and work outing the math that makes this possible. Other strategies may include mutating the crypto object (which I forgot to freeze within the obfuscation...), but I consider that cheating. Good luck.
